PDF Scraping using Python

August 9, 2022

Now a day’s medical field is advancing use of Artificial intelligence as deep learning and computer vision for increased use of document extraction from medical documents to automate patient reports. The tasks should be automated as much as possible to enable efficiency. Standardizations are very important to support automation. By having standard processes and tools, a global maintenance and enhancements of tools can be provided. Then all users can benefit from this as updates could be made available to all users through standard processes.

The project aims to increase the understanding of developing a framework for information extraction (IE) from unstructured PDF documents to CSV files. As a result we have a tool to neatly extract tabular data from PDF to CSV files. This project will aim to develop a generic tool to develop a patient narrative using pre filled word document template.

There are several Python packages that make it simple to convert PDF to CSV files. In the below methods, we’ll look at how to use five different modules and methods to convert PDF to CSV in Python. All five methods were carefully chosen and implemented to determine which produces the best results in terms of efficiency and accuracy. When we parse a PDF document using Python, it does not recognise the rows and columns and sometimes fails to perform well, producing incorrect results. Challenges were encountered while extracting the data, such as inconsistency with headers on each page and no border line differentiation for pdf tables. I’ve found that method 3 using Camelot works best for these kinds of inconsistencies in documents, especially if they’re scanned or have no margins. Evaluating extracted document accuracy shows Camelot gives 98% accuracy

The output document template provided is in a *.docx format with 3 pages in total. Following the review of QC comments, we have 13 appendices from which the data will be loaded into the template to generate the patient narrative. The study demonstrates how to use Python to automate the filling of blanks in word documents. The project requires data extraction from unstructured tables, which will require text mining before information extraction, as well as manual template editing to replace blanks in the template with placeholders with curly brackets. So, after creating csv files, our first step was to manually update our template with placeholders. After we prepared a template with placeholders. The next step is to go through the template paragraph by paragraph, using Python Pandas and text mining techniques to update the placeholders.

Pre-processing and text mining 14 pandas data frames, which will then be used to pull dictionary terms in docxtpl context to replace template placeholders. Panda data frames are pre-processed to extract information from csv files in accordance with QC comments in the Narrative template. Each paragraph of the template narrative includes comments that will guide you through the process of extracting required csv files and pre-processing columns of each csv file in pandas as needed. The narrative template placeholders and code required in dictionary code to update the output values by extracting information from pre-processed csv files to patient narrative for patient The dictionary code in docxtpl used for context has used the traditional representation of using loc and iloc systems. Where the methodology locks specific data specified in code and presents data that satisfies the conditionals. For example, in the age (X) methodology, we assign placeholders in the narrative template to the extraction code. Assume we assign age placeholder “X” by using age data frame with only subject identifier.

The comparison and evaluation have been carried out for patient narrative that was accurately pulling up the data using place holders and dictionary approach with the use of docxtpl module. This Project designed a system architecture for the diverse-structure PDF tabular data document extraction. The information has been extracted from PDF to CSV using different methods. Camelot has worked well for PDF to CSV tabular data extraction. It was programmed in Python, a cross-platform programming language. The program was modularized using pandas’ data frame functions to transform the extracted dataset in CSV. At the end of the extraction of PDF to CSV, Camelot method for information extraction has been evaluated for parsing report that gave 98% of accuracy. After the PDF document extraction from PDF to CSV files this project designed a patient narrative tool in Python. For designing patient narrative tool of single patient, we have using classic approach of pandas reading csv files and transforming the data to extract required information. Initially project has dealt the major part with researching best method for data extraction and minor work has been done with generating one patient narrative with classical approach of updating placeholders in narrative template document. Later, the narrative for another patient has been developed for quality check purposes. So, this project has dealt with Information extraction and developing two patient narratives.

We learned a lot about how we can apply python solutions within our clinical environment. We had been able to help some colleagues in PDF extraction. By using python, the hurdles to perform PDF to CSV conversion from the scanned and unstructured document has taken more than 80% of project implementation time while 20% was given for creating patient narrative tool. Therefore, with python understanding we also need to learn the template language to create narratives. Hence forth, I am more confident now for more opportunities with Unstructured PDF extraction to CSV files as well as clear picture of all the requirement that for template filling using python from CSV files. Looking forward for more opportunities in future
